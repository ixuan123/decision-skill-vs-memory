# Skill vs Memory 决策器

## 触发条件

- 完成一个任务后，问自己"这个要不要保存"
- 不确定存 Skill 还是 Memory 时
- 清理/整理经验时

---

## 决策流程

```
任务完成
   │
   ▼
可重复使用？ ── 否 ──→ 存 Memory
   │
   是
   │
   ▼
别人也能用？ ── 否 ──→ 存 Memory
   │
   是
   │
   ▼
流程固定？ ── 否 ──→ 存 Memory
   │
   是
   │
   ▼
  存 Skill ✅
```

---

## 详细判断规则

### 存 Skill 的情况

| 类型 | 例子 |
|------|------|
| **流程类** | 打开 Chrome CDP 读取抖音 |
| **工具类** | 安装 Tavily API |
| **方法论** | 横纵分析法、内容创作 |
| **自动化脚本** | 股票监控、搜索 |
| **可复用框架** | 自我改进、主动服务 |

**特征：**
- 步骤固定，可以写成文档/脚本
- 下次遇到类似任务能直接用
- 别人也能用

### 存 Memory 的情况

| 类型 | 例子 |
|------|------|
| **配置信息** | API Key、端口、路径 |
| **规则/约束** | "不能重启 gateway" |
| **一次性操作** | 删除重复 skill |
| **教训/踩坑** | Windows 控制台编码问题 |
| **个人偏好** | 用户喜欢什么风格 |
| **临时状态** | 当前项目进度 |

**特征：**
- 个人化的、独特的
- 不重复发生的
- 只是记录信息

---

## 决策树（快速版）

```
这个任务下次还会遇到吗？
  ├─ 不会 → Memory
  └─ 会 → 别人能用吗？
            ├─ 不能 → Memory
            └─ 能 → 步骤固定吗？
                      ├─ 不固定 → Memory
                      └─ 固定 → Skill ✅
```

---

## 操作

### 创建 Skill
```
skill_workshop(action="create", name="xxx", description="xxx", proposal_content="完整步骤")
```

### 写入记忆
```
memory/YYYY-MM-DD.md  # 当日记录
MEMORY.md             # 长期记忆
```

### 不确定时
两个都存，以后精简。

---

## 实战例子

| 任务 | 决策 | 原因 |
|------|------|------|
| Chrome CDP 读抖音 | Skill | 步骤固定，可复用 |
| Tavily API Key | Memory | 配置信息，非流程 |
| 删除重复 skill | Memory | 一次性操作 |
| "不要重启" | Memory | 规则类 |
| 横纵分析法 | Skill | 方法论，可复用 |
| 编码踩坑 | Memory | 教训，不重复 |

- 定期回顾 Skill 列表
- 长期不用的 Skill 标记为休眠
- 过时的 Skill 更新或删除
