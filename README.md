# decision-skill-vs-memory

> 🧠 一个帮 AI 做选择的工具：完成一件事后，经验该存成 **"技能"（Skill）** 还是 **"笔记"（Memory）**

---

## 一句话解释

AI 每天干很多活，干完活得记住经验。但记住的地方有两个：

| 地方 | 人话 | 类比 |
|------|------|------|
| **Skill（技能）** | 能复用的方法 | 菜谱——下次照着做 |
| **Memory（记忆）** | 一次性的信息 | 日记——就记这一次 |

这个 Skill 就是帮你判断：**这个经验，该放哪个地方？**

---

## 为什么需要判断？

### 存错了会怎样？

**情况1：该记的没记**

```
干了一件事 → 存在 Memory 里
下次遇到同样的事 → 忘了怎么干的 → 重新学一遍
```
结果：**重复劳动，越干越蠢**

**情况2：不该记的记了**

```
一个临时操作 → 存成 Skill
仓库越来越大 → 全是没用的东西
找东西越来越难
```
结果：**仓库臃肿，全是垃圾**

---

## 判断方法（3 个问题）

每次完成任务，问自己 3 个问题：

```
问题1：下次还会遇到吗？
  └─ 不会 → 记笔记（Memory）
  └─ 会 → 下一步

问题2：别人也能用吗？
  └─ 不能（专属的） → 记笔记（Memory）
  └─ 能 → 下一步

问题3：步骤是固定的吗？
  └─ 不固定（每次不一样） → 记笔记（Memory）
  └─ 固定 → 存技能（Skill）✅
```

## 举几个例子

| 你干的事 | 存哪？ | 为什么 |
|----------|--------|--------|
| 配置了 Tavily 的 API Key | 📝 笔记 | 这只是个密码，不是方法 |
| "父亲大人不喜欢重启" | 📝 笔记 | 这是规矩，不是技能 |
| 删除了 5 个重复的 skill | 📝 笔记 | 一次性的，下次不会再来 |
| Chrome CDP 读抖音 | ✅ 技能 | 步骤固定，下次还能照着做 |
| 横纵分析法 | ✅ 技能 | 方法论，通用的 |
| Windows 编码踩坑 | 📝 笔记 | 教训而已，不是技能 |

---

## 记住一句话

> **技能是"怎么做"，笔记是"发生了什么"。**
>
> - 技能教我下次怎么做
> - 笔记让我记住这次经历了什么
